如鹏网《Javaweb》面试题

1、

说说servlet工作的原理

解析：

（1）服务器启动时解析web.xml，把servlet的虚拟路径管理起来

（2）服务器收到浏览器的请求后，就把请求解析为request对象，同时创建一个response对象

（3）服务器根据请求路径找到匹配的servlet，调用servlet对应的doGet或者doPost方法，同时把request和response传递给该方法

（4）servlet就开始对请求进行处理，处理完成后生成响应发送到浏览器


 2、

GET和POST提交方式的区别

解析：

（1）GET方式的请求把请求参数拼接在请求路径后面，在有些时候（如超链接上）特别方便，但这种方式所能携带的数据量有限，并且可以被直接看到，不太安全

（2）POST方式把请求参数放在请求体中，可以携带大量数据，并且安全性相对较高

request.getAttribute()和request.getParameter()有何区别？

 4、

jsp有哪些内置对象?作用分别是什么?分别有什么主要方法？

解析：

JSP共有以下9个内置的对象：

（1）pageContext 页面上下文对象，可以通过该对象获取其他内置对象

（2）request 请求对象，包含所有客户端请求信息

（3）session session对象

（4）application  ServletContext对象（前四个是非常重要的web域对象，用来作为数据仓库存储不同生命周期的数据使用）

（5）response 响应对象

（6）out 输出流对象，用来把响应输出到客户端

（7）config   ServletConfig对象

（8）page  JSP页面本身，即this

（9）exception 异常对象，如果此JSP页面被设置为错误页面，则此对象可用，表示发生的异常

 8、

说出数据连接池的工作机制是什么？

解析：

（1）程序启动时连接池就创建好若干数据库连接并保存起来

（2）当用户需要连接时，直接从连接池中获取连接

（3）当用户本次使用连接完毕后，就把连接归还给连接池

（4）假设在没有空闲连接时用户A来获取连接，如果当前连接数小于最大连接数，则连接池创建新的连接并交给A；如果当前连接数达到最大连接数，则连接池不再创建新的连接，而是等待其他用户归还连接，再把归还的连接交给A



 9、

购物车和订单的具体实现？

解析：

（1）购物车信息保存在session中，包含若干个购物车项信息，每个购物车项就是用户购买的某个商品的信息以及购买数量

（2）订单信息根据购物车生成，再加上收货地址等信息。订单中包含若干订单项，订单项即对应购物车中的购物车项，也是用户购买的商品信息以及数量

 13、

java servlet中forward和sendRedirect的区别是什么？

解析：

forward是服务器内部的控制权转移，服务器和浏览器之间只进行一次通讯，浏览器地址栏不发生变化；

sendRedirect是让浏览器去重定向，服务器和浏览器之间会再进行一次通讯，浏览器地址栏会发生变化；

 48、

说一下Servlet的的生命周期。

解析：

Servlet被服务器实例化后，容器运行其init方法，请求到达时运行其service方法，service方法自动运行与请求对应的doGet或者doPost方法，当服务器决定将实例销毁的时候调用其destroy方法。
